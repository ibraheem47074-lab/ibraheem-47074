Like Count Fix Verification

The Bug (Before Fix)

Problem: The old code counted ALL guest likes from session:

foreach ($_SESSION as $key => $value) {
    if (strpos($key, 'guest_likes_') === 0 && $value === true) {
        $guest_likes++; // This adds ALL likes!
    }
}

Result: Article 1 shows 1 like, Article 2 shows 2 likes, Article 3 shows 3 likes (cumulative)

The Fix (After Fix)

Solution: The new code only counts likes for the specific article:

$guest_likes = 0;
$session_key = 'guest_likes_' . $news_id;
if (isset($_SESSION[$session_key]) && $_SESSION[$session_key] === true) {
    $guest_likes = 1; // Only this article's like
}

Result: Each article shows its own individual like count

Fix Summary

✅ FIXED

  • Root Cause: Guest like counting logic was counting ALL session likes instead of per-article likes
  • Fix Applied: Changed counting logic to only check the specific article's session key
  • File Modified: api/toggle_like.php lines 67-72
  • Result: Each article now shows its own individual like count

Note: The fix only affects guest users (non-logged-in users). Registered users were already working correctly with database-based counting.
